# SBOM Comparison Tool

A tool for evaluating the accuracy of Software Bill of Materials (SBOM) generation tools for Rust projects. It compares the dependency information extracted from a project's `cargo metadata` output against the components listed in a generated SBOM, and writes the results to an Excel report.

---

## Background

Four SBOM generation tools are evaluated in this study:

| Tool | Output filename |
|---|---|
| Syft | `syft.json` |
| Trivy | `trivy.json` |
| Cargo SBOM | `cargosbom.json` |
| CycloneDX Rust Cargo | `<package-name>.cdx.json` |

`cargo metadata` is used as the ground truth because it is produced directly by the Rust toolchain and reflects the authoritative, fully resolved dependency graph of the project.

---

## What is Compared

For each SBOM file, the tool computes the following metrics:

| Metric | Description |
|---|---|
| Total `cargo metadata` packages | Number of packages in the ground truth, including transitive dependencies |
| Total SBOM components | Number of components reported by the tool |
| Correct packages | Packages present in both `cargo metadata` and the SBOM |
| Missed packages | Packages in `cargo metadata` but missing from the SBOM |
| Hallucinated packages | Packages in the SBOM but not present in `cargo metadata` |
| Version mismatches | Packages with matching names but different versions |
| Correct transitive dependencies | Transitive dependency relationships correctly represented in the SBOM |
| Missing transitive dependencies | Transitive dependency relationships in `cargo metadata` absent from the SBOM |
| False transitive dependencies | Transitive dependency relationships in the SBOM not present in `cargo metadata` |

## Project Structure

```
project/
├── path.txt               # List of projects and their SBOM folders
├── sbom_results.xlsx      # Generated output (created on first run)
└── src/
    ├── main.py            # Reads path.txt and runs comparisons
    ├── cargo.py           # Parses cargo metadata JSON
    ├── sbom.py            # Parses CycloneDX SBOM JSON
    ├── comparator.py      # Runs the comparison and writes the Excel report
    └── package.py         # Defines the Package class used across all modules
```

## Requirements

- Python 3.8 or later
- The following Python packages:

```bash
pip install openpyxl
```

- Each project must have:
  - A `metadata.json` file generated by `cargo metadata` (see [Generating Ground Truth](#generating-ground-truth))
  - A folder named `sbom/` containing the SBOM JSON files to evaluate

---

## Generating Ground Truth

For each Rust project, generate the `cargo metadata` file using:

```bash
cargo metadata --locked --format-version 1 > metadata.json
```

The `--locked` flag ensures that the exact dependency versions recorded in `Cargo.lock` are used, making the output reproducible.

---

## Configuration

### 1. Set the base path

Open `main.py` and set `BASE_PATH` to the folder containing all your Rust projects:

```python
BASE_PATH = "<path>"
```

### 2. Edit `path.txt`

Each line specifies one project in the following format:

```
<project_folder> | <sbom_subfolder>
```

`BASE_PATH` is automatically prepended to both sides, and `/metadata.json` is automatically appended to the left side. Lines starting with `#` and empty lines are ignored.

**Example `path.txt`:**

```
# Format: <project_folder> | <sbom_subfolder>

# Top 20 projects
claw-code/rust  | claw-code/rust/sbom
rustdesk        | rustdesk/sbom
deno            | deno/sbom
tauri           | tauri/sbom
```

---

## Usage

```bash
python3 main.py
```

The tool will:
1. Read all project paths from `path.txt`
2. For each project, run a comparison against every `.json` file in the `sbom/` folder
3. Write all results to `sbom_results.xlsx`

---

## Output

Results are written to `sbom_results.xlsx`. Each row represents one SBOM file comparison and includes all metrics listed in the [What is Compared](#what-is-compared) section.
